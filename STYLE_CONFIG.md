# 工程笔记风格配置

> 此文件用于指导 AI 将自然语言日志转换为 markdown 格式。请严格遵循以下规则，保持所有笔记风格统一。

---

## 1. 日期格式

- **标准格式**：`YYYY.M.D HH：MM`（使用英文句点分隔年月日，全角冒号分隔时分）
  - 示例：`2025.11.3 20：51`
- **仅日期**（无具体时间）：`YYYY.M.D`
  - 示例：`2026.3.15`
- **日期范围**：`YYYY.M.D-YYYY.M.D` 或 `YYYY.M.D-M.D`
  - 示例：`2025.11.20-2025.11.23`、`2026.3.6-3.10`
- **日期 + 描述**（无具体时间）：`YYYY.M.D 描述内容`
  - 示例：`2025.11.15 备考/考试`
- 日期行独立成行，顶格书写，不作为 markdown 标题（不使用 `#`）

## 2. 分隔规则

- **日期之间**：每两个不同日期之间插入 `---`（三条横线）作为分隔线
- **同一天内**：不同时间段（如 `13：04`、`21：58`）之间用空行分隔，不插入 `---`
- **同一天内**的不同条目之间也用空行分隔

## 3. 内容格式

- 主要条目使用**有序列表**（`1.` `2.` `3.` ...）
- 子条目或补充说明使用无序列表或直接另起段落
- 列表项之间保留空行
- 纯文字反思/总结不需要编号，直接书写段落
- 特殊标签如「总结问题：」「技术路线：」「需求：」等可以独立成行，后跟列表或段落

## 4. 语言风格

- **主语言**：简体中文
- **技术术语**：保留英文原名（如 PID、CAN、FreeRTOS、ROS2、FPGA、IMU 等）
- **语气**：个人笔记风格，口语化、直接，可以带有情绪表达
- **人称**：第一人称「我」，偶尔省略主语

## 5. 特殊元素

- **链接**：保留原始 URL，使用 markdown 链接格式 `[标题](URL)`
- **代码/文件名**：使用反引号包裹，如 `main.c`、`BubinGimbalV2`
- **时间子标记**：同一天内的不同时刻用 `HH：MM` 独立一行标注（不视为新的一天）
- **元数据**：如「总学习时长：Xh」可附加在日期行末尾

## 6. 转换规则（自然语言 → Markdown）

当用户输入自然语言日志时，按以下规则处理：

1. **识别日期**：提取日期和时间信息，统一为 `YYYY.M.D HH：MM` 或 `YYYY.M.D` 格式
2. **分条**：将描述性内容拆分为有序列表项，每条一个要点
3. **保留原意**：不改变用户原文的意思、措辞和语气
4. **保留情绪**：emoji、语气词、口语化表达均保留
5. **链接处理**：用户提到的资料名/视频名尽量保留并尝试转为 markdown 链接
6. **术语不翻译**：PID、CAN、SPI、I2C、DMA、FreeRTOS、ROS 等技术缩写保持原样
7. **分隔线**：自动在相邻两天之间插入 `---`

## 7. 示例

### 输入（自然语言）：
```
11月3号晚上8点51，今天把标准库转hal库推进到uart了，初步了解了一下can协议。明确了学习规划，先扎实stm32基础然后上手freertos。当前学习以中科大的rm视频为主。

11月4号晚上8点半，今天课多进度慢，复习了stm32时钟系统和cubemx配置时钟。
```

### 输出（markdown）：
```
2025.11.3 20：51

1. 在标准库转hal库过程，今日已推进至uart

2. 初步了解can协议

3. 明确了大致学习规划，先扎实stm32基础（包括熟悉hal库和提升独立开发能力），然后上手freertos

4. 当前学习以中科大的rm视频为主

---
2025.11.4 20：30

1. 今天有较多的课，所以进度几乎未推进，考虑到十五号的期中考试，日后进度可能缓慢

2. 复习了stm32系列时钟系统和底层原理，熟悉了在cubemx中配置时钟的流程
```
